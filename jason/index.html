Python · agents · systems design

corp-engine

An operating system for agent-operated business entities. Every entity is a git repository: schema-validated JSON state, markdown runbooks an AI operator can follow, and dependency-free HTML dashboards generated straight from that state. The engine runs itself as its own first tenant and hosts others, including a trading entity that files its own operations reports. It's my long-running experiment in how much of a business can be made legible enough for an agent to operate, and how you test a system whose operator is a language model.

How I build

Every project above shares the same spine: test-driven development against a four-layer test architecture (unit, integration, functional, and contract) with Pact contract testing wherever two systems shake hands, and CI/CD that deploys on green. The stack varies (TypeScript, Python, Lua, Swift); the discipline doesn't.
